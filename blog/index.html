自定义域名

注册域名
可自行阿里云或其他机构注册域名

域名解析配置

域名配置最常见有2种方式，CNAME和A记录，
A记录就是把一个域名解析到一个IP地址（Address，特制数字IP地址）
而CNAME记录就是把域名解析到另外一个域名。

4.关于hexo

是一个快速、简单和强大的博客框架。 你用 Markdown (或其他标记语言)写文章，Hexo 在几秒钟内生成带有漂亮主题的静态文件。

安装
$ npm install -g hexo-cli
or
$ npm install hexo

安装Hexo后 运行以下命令初始化

$ hexo init <folder>
$ cd <folder>
$ npm install

一旦初始化，这是你的项目文件夹的样子:

.
├── _config.yml    // 站点配置文件。您可以在这里配置大多数设置。
├── package.json   // 应用程序数据 安装包设置文件
├── scaffolds      // 脚手架文件夹。创建新文章时，Hexo 将新文件置于脚手架上。
├── source         // 源文件夹。 这就是你放网站内容的地方
|   ├── _drafts   
|   └── _posts
└── themes        // themes 主题

生成文件

$ hexo generate # 生成静态文件

启动一个本地服务器 

$ hexo server -p 端口号

部署

$ hexo deploy

清除缓存文件

$ hexo clean
常见命令
hexo new "postName" #新建文章
hexo new page "pageName" #新建页面
hexo generate #生成静态页面至public目录
hexo server #开启预览访问端口（默认端口4000，'ctrl + c'关闭server）
hexo deploy #部署到GitHub
hexo help  # 查看帮助
hexo version  #查看Hexo的版本

// 缩写
hexo n == hexo new
hexo g == hexo generate
hexo s == hexo server
hexo d == hexo deploy

hexo s -g #生成并本地预览
hexo d -g #生成并上传
tips

1._config.yml 配置主题

_config.yml 配置的主题名称需要和themes下一致，不然会报错

2.缺少hexo插件 

$ npm ls --depth 0  // 查看hexo插件安装情况

3.Hexo CNAME文件在每次部署后删除了 

CNAME文件放置在source目录下
